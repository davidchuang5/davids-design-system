# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This repo has two distinct parts:

1. **The design system package** (root) — a publishable React component library built with Vite, Vitest, and Storybook.
2. **The agent pipeline** (`.claude/`) — a multi-agent pipeline using Claude Code's native agent system to generate components, tests, and stories automatically.

---

## Commands

### Design system (root)

```bash
npm install
npm run build          # tsc + vite library build → dist/
npm run test           # vitest run (single pass)
npm run test:watch     # vitest in watch mode
npm run lint           # eslint src/ (TS/TSX)
npm run typecheck      # tsc --noEmit
npm run dev            # Storybook on localhost:6006
npm run build-storybook
```

### Token JS export script (no LLM call)

```bash
npx tsx scripts/generate-tokens.ts src/tokens/tokens.json src/tokens
```

Converts `tokens.json` → `tokens.js` + `tokens.d.ts`. Called automatically by the `/generate-design-system` skill after Phase 1.

### Agent pipeline

In a Claude Code conversation in this repo, run:

```
/generate-design-system
```

To change the component list, color palette, or border radius, edit the **Design System Spec** section at the top of `.claude/skills/generate-design-system/SKILL.md`.

## Architecture

### Design system package (root)

- **Entry**: `src/index.ts` — barrel export, auto-updated by the pipeline after each run.
- **Tokens**: `src/tokens/` — `variables.css` (CSS custom properties), `tokens.json` (raw values), `tokens.js` / `tokens.d.ts` (JS exports generated by `scripts/generate-tokens.ts` — no LLM call).
- **Components**: `src/components/<Name>/` — each folder contains `<Name>.tsx`, `<Name>.module.css`, `<Name>.d.ts`, `<Name>.test.tsx`, `<Name>.stories.tsx`.
- **Build**: Vite library mode outputs `dist/index.js` (ESM), `dist/index.cjs` (CJS), `dist/index.d.ts`, and `dist/styles.css`. React and react-dom are external peers.
- **CSS token alias**: `@tokens` resolves to `src/tokens` (configured in `vite.config.ts`).
- **Component conventions**: `forwardRef`, `displayName`, props extending native HTML element attributes, `size` (sm/md/lg) and `variant` props, WAI-ARIA attributes, CSS Modules using only `--ds-*` custom properties.

### Agent pipeline (native Claude Code)

The pipeline is a skill invoked via `/generate-design-system`. It orchestrates four subagents defined in `.claude/agents/`:

| Phase | Agent | Model | Tools |
|---|---|---|---|
| 1 — Tokens | `token-agent` | haiku | `Read`, `Write` |
| Checkpoint | Human confirmation | — | — |
| 2 — Component | `component-agent` | sonnet | `Read`, `Write`, `Glob` |
| 3 — Tests + Stories (parallel) | `test-agent` + `storybook-agent` | sonnet | `Read`, `Write`, `Glob` |

Key behaviors:
- **Idempotent**: components with an existing folder under `src/components/` are skipped. Delete the folder to force regeneration.
- **Token phase is also idempotent**: skipped if `src/tokens/variables.css` already exists.
- **After completion**: `src/index.ts` barrel exports are updated for all newly built components.
- **`SRC_DIR`** is `/Users/davidchuang/enterprise-design-system/src` — update in SKILL.md if the repo moves.

### Agent definitions

Each subagent is a markdown file in `.claude/agents/` with frontmatter (name, model, color, tools) and a system prompt body. The skill's orchestration instructions reference agents by name; Claude Code's Task tool spawns them as subprocesses.
