## Badge

### Overview

The Badge component is a small, labeled element used to highlight, categorize, or draw attention to information. Badges typically appear inline or attached to other elements to provide supplementary context such as status, count, or category labels. Use badges for notifications, status indicators, labels, or to mark new/updated content.

### Usage

```jsx
import { Badge } from './Badge'

export default function BadgeExample() {
  return (
    <div style={{ display: 'flex', gap: '8px', alignItems: 'center' }}>
      <span>Orders:</span>
      <Badge variant="primary" size="md">
        12 new
      </Badge>
    </div>
  )
}
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'primary' \| 'success' \| 'warning' \| 'danger'` | `'default'` | Visual style variant that indicates status or importance. |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Size of the badge. |
| `className` | `string` | — | Additional CSS classes to apply to the badge. |
| `children` | `React.ReactNode` | — | Content displayed inside the badge. |

The Badge component also extends all standard HTML span attributes.

### Accessibility

- **Semantic HTML**: The Badge renders as a `<span>` element, which is semantically neutral. For status badges that convey important information, consider wrapping or complementing with appropriate ARIA attributes or label text.
- **Color Contrast**: Each variant meets WCAG AA standards for color contrast against its background. Do not rely solely on color to convey information; use text labels as well.
- **Screen Readers**: Badge content is read aloud by screen readers. Ensure the text content is clear and self-contained. For icon-only or purely visual badges, provide `aria-label` or `aria-describedby` to explain the badge's purpose.
- **Focus Management**: The Badge component does not receive keyboard focus by default. If interactivity is needed, wrap it in a button or link element.

### Do / Don't

**Do:**
```jsx
// Use badges to label or highlight short amounts of text
<div>
  <h3>
    New Feature
    <Badge variant="primary" size="sm">
      Beta
    </Badge>
  </h3>
  <p>This feature is currently in beta testing.</p>
</div>
```

**Don't:**
```jsx
// Don't use badges for long content or as interactive elements
<Badge>
  This is a very long description that should be in a separate component
  because badges are meant for short, scannable labels only.
</Badge>

// Don't rely only on color to convey meaning
<Badge variant="danger">12</Badge> {/* Unclear what the number represents */}
```
