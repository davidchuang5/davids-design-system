# Badge

## Overview

Badge is an inline status label used to call out short, categorical information such as counts, states, or labels. It supports multiple semantic variants and three sizes, all drawn from design tokens. Badges are commonly used to highlight status, add counts to navigation items, or label categorical data without taking up significant space.

## When to Use

- Display status information (success, warning, error, pending)
- Show notification counts on buttons or navigation items
- Label categories or tags
- Highlight metrics or key data
- Add visual emphasis to short text labels

## Usage

```jsx
import { Badge } from '@puma-ui/components'

export function BadgeExample() {
  return (
    <div style={{ display: 'flex', gap: '8px', flexWrap: 'wrap' }}>
      <Badge>New</Badge>
      <Badge variant="success">Approved</Badge>
      <Badge variant="warning">Pending</Badge>
      <Badge variant="danger">Rejected</Badge>
      <Badge variant="secondary">Draft</Badge>
      <Badge variant="outline">Available</Badge>
    </div>
  )
}
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'primary' \| 'secondary' \| 'success' \| 'warning' \| 'danger' \| 'outline'` | `'default'` | Visual style of the badge. Choose a variant that matches the semantic meaning of the content. |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Size of the badge. Use `sm` for tight spaces, `lg` for prominent labels. |
| `aria-label` | `string` | — | Accessible label override. Use when badge text alone is not descriptive enough for screen-reader users (e.g. a numeric count without context). |
| `className` | `string` | — | Additional CSS class names for custom styling. |
| `role` | `string` | `'status'` | ARIA role. Use `'status'` for dynamic updates, `'img'` for decorative badges with aria-label. |
| `ref` | `Ref<HTMLSpanElement>` | — | Forward ref to the underlying span element. |

All standard HTML span attributes are also supported (e.g., `id`, `data-*`, event handlers).

## Variant Examples

### Semantic Variants

Use variants to communicate meaning:

```jsx
<Badge variant="default">Default</Badge>
<Badge variant="primary">Primary</Badge>
<Badge variant="secondary">Secondary</Badge>
<Badge variant="success">Success</Badge>
<Badge variant="warning">Warning</Badge>
<Badge variant="danger">Danger</Badge>
<Badge variant="outline">Outline</Badge>
```

**Variant Usage Guidelines:**
- `default`: Neutral information with no special meaning
- `primary`: Featured or highlighted information
- `secondary`: Less prominent, supporting information
- `success`: Positive state or completed action
- `warning`: Caution or attention-required state
- `danger`: Error, critical, or failed state
- `outline`: Secondary style with border, lower visual weight

## Size Examples

### Small, Medium, Large

```jsx
<Badge size="sm">Small</Badge>
<Badge size="md">Medium</Badge>
<Badge size="lg">Large</Badge>
```

**Size Usage Guidelines:**
- `sm`: Use in dense layouts or when space is limited (e.g., table cells)
- `md`: Default size for most use cases
- `lg`: Use when badges are prominent focal points or in large cards

### Size with Variants

```jsx
<Badge variant="success" size="sm">
  3
</Badge>
<Badge variant="warning" size="md">
  In Progress
</Badge>
<Badge variant="danger" size="lg">
  Failed
</Badge>
```

## Accessibility

### Screen Reader Behavior

- Badge renders as a `<span role="status">` by default, which announces dynamic content updates without moving focus
- For decorative badges that represent a concept, use `role="img"` with an `aria-label`

### Keyboard Interaction

- Badges themselves are not keyboard interactive—they are informational elements
- If a badge wraps a clickable element, ensure the interactive element (button, link) handles keyboard navigation

### ARIA Labels

Always provide an `aria-label` when the visible text may be ambiguous or insufficient:

```jsx
// Good: Number badge with context
<Badge aria-label="3 new messages">3</Badge>

// Good: Decorative badge with aria-label
<Badge role="img" aria-label="status: active" variant="success" />

// Avoid: Ambiguous text without label
<Badge>3</Badge>
```

### Best Practices

- Keep badge text short and meaningful
- Use semantic variants to reinforce meaning (color alone is not sufficient for color-blind users)
- Pair colors with text or icons when communicating critical information
- Test with screen readers to ensure context is clear

## Do / Don't

### Do

```jsx
// Correct: Clear label with semantic variant
<Badge variant="success" aria-label="2 items approved">
  2
</Badge>

// Correct: Using variant to reinforce meaning
<Badge variant="danger">Error</Badge>

// Correct: Descriptive text in badge
<Badge variant="warning">Pending Review</Badge>
```

### Don't

```jsx
// Incorrect: Ambiguous numeric badge without aria-label
<Badge>5</Badge>

// Incorrect: Relying on color alone for critical information
<Badge variant="danger" />

// Incorrect: Using badge for interactive content
<Badge onClick={handleClick}>Click me</Badge>
// Use a Button component instead for interactive elements
```
