title: Accordion
description: A collapsible panel component for organizing content into expandable sections.

## Overview

Accordion displays a list of collapsible panels where users can reveal and hide content sections. Use Accordions to organize large amounts of related information, reduce visual clutter, or create progressive disclosure flows. The component supports both single and multiple expansion modes, controlled and uncontrolled state management, and full keyboard navigation.

## Usage

```jsx
import { Accordion } from '@/components/Accordion'

const items = [
  {
    id: 'faq-1',
    header: 'What is this product?',
    content: 'It is a comprehensive accordion component for organizing content.',
  },
  {
    id: 'faq-2',
    header: 'How do I use it?',
    content: 'Simply pass an array of items with id, header, and content properties.',
  },
  {
    id: 'faq-3',
    header: 'Is it accessible?',
    content: 'Yes, it includes full keyboard navigation and ARIA attributes.',
  },
]

export function FAQAccordion() {
  return (
    <Accordion
      items={items}
      selectionMode="single"
      defaultExpandedIds={['faq-1']}
      variant="default"
      size="md"
    />
  )
}
```

## Single vs Multiple Mode

By default, Accordion operates in `selectionMode="single"` where only one panel can be open at a time. Switch to `selectionMode="multiple"` to allow multiple panels to be open simultaneously.

```jsx
<Accordion
  items={items}
  selectionMode="multiple"
  defaultExpandedIds={['faq-1', 'faq-3']}
/>
```

## Controlled vs Uncontrolled

Use `defaultExpandedIds` for uncontrolled state (component manages expansion internally):

```jsx
<Accordion items={items} defaultExpandedIds={['faq-1']} />
```

For controlled state, provide both `expandedIds` and `onExpandedChange`:

```jsx
const [expanded, setExpanded] = useState(['faq-1'])

return (
  <Accordion
    items={items}
    expandedIds={expanded}
    onExpandedChange={setExpanded}
  />
)
```

## Variants

The component offers three visual variants:

```jsx
// Default variant with separator borders
<Accordion items={items} variant="default" />

// Bordered variant with full border around each item
<Accordion items={items} variant="bordered" />

// Ghost variant with minimal visual treatment
<Accordion items={items} variant="ghost" />
```

## Size Options

Control padding and font size inside panels:

```jsx
<Accordion items={items} size="sm" />  {/* Compact */}
<Accordion items={items} size="md" />  {/* Default */}
<Accordion items={items} size="lg" />  {/* Spacious */}
```

## Collapsible Behavior

By default in `selectionMode="single"`, the open item cannot be collapsed by clicking it again. Set `collapsible={true}` to allow the last open item to be collapsed:

```jsx
<Accordion items={items} selectionMode="single" collapsible={true} />
```

## Keyboard Interactions

- **Enter** or **Space**: Expand/collapse the focused item
- **ArrowDown**: Move focus to the next trigger button (wraps to first)
- **ArrowUp**: Move focus to the previous trigger button (wraps to last)
- **Home**: Move focus to the first trigger button
- **End**: Move focus to the last trigger button

All keyboard interactions are automatically supported. Items can be disabled with the `disabled` property on an AccordionItem.

## Accessibility

The Accordion component implements the [WAI-ARIA Accordion Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/accordion/):

- **Role**: Root container has `role="list"`, each panel trigger has `role="button"`
- **aria-expanded**: Set to `true` when a panel is open, `false` when closed
- **aria-controls**: Connects each trigger to its controlled panel via `aria-controls={panelId}`
- **aria-labelledby**: Each panel region is labeled by its trigger
- **aria-disabled**: Set on disabled items to announce disabled state
- **aria-posinset/aria-setsize**: Communicates position within the set for screen readers
- **Hidden attribute**: Closed panels are marked `hidden` to remove them from the accessibility tree

Headers and content can contain any React nodes, including semantic HTML elements. Buttons and interactive elements inside content remain accessible.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `AccordionItem[]` | — | List of panels to render. Each item has `id` (string), `header` (ReactNode), `content` (ReactNode), and optional `disabled` (boolean). |
| `selectionMode` | `'single' \| 'multiple'` | `'single'` | Whether one or many items can be expanded at the same time. |
| `defaultExpandedIds` | `string[]` | `[]` | IDs of items expanded on first render. In single mode only the first ID is used. |
| `expandedIds` | `string[]` | — | Controlled set of expanded item IDs. Must be paired with `onExpandedChange`. |
| `onExpandedChange` | `(expandedIds: string[]) => void` | — | Callback fired whenever the expanded set changes. Required when using controlled state. |
| `variant` | `'default' \| 'bordered' \| 'ghost'` | `'default'` | Visual style of the accordion. |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Padding and font size inside panels. |
| `collapsible` | `boolean` | `false` | When true, the open item can be collapsed by clicking again (single mode only). |
| `className` | `string` | — | Additional CSS class names to apply to the root element. |

## Do / Don't

### Do: Use Accordion for Related Content Groups

```jsx
// Good: Accordion organizes FAQs into logical sections
<Accordion
  items={[
    { id: 'billing', header: 'Billing', content: 'Questions about billing...' },
    { id: 'shipping', header: 'Shipping', content: 'Questions about shipping...' },
    { id: 'support', header: 'Support', content: 'Questions about support...' },
  ]}
  selectionMode="single"
/>
```

### Don't: Overload Accordion with Unrelated Content

```jsx
// Avoid: Using Accordion for every content section regardless of relationship
<Accordion
  items={[
    { id: 'header-nav', header: 'Navigation', content: 'Site navigation items' },
    { id: 'random', header: 'Random Facts', content: 'Unrelated content' },
    { id: 'ads', header: 'Advertisements', content: 'Promotional content' },
  ]}
/>
```
