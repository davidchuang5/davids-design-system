# Button

## Overview

The Button component is a reusable, accessible control for triggering actions or navigating within the application. It supports three visual variants (primary, secondary, ghost), three sizes (sm, md, lg), optional icon slots, and a loading state. All styling is driven by CSS custom properties, making it easy to theme across the design system.

Use buttons for actions that change application state, submit forms, or trigger workflows. For navigation to a new page or external URL, use a link styled as a button instead.

## Usage

```jsx
<Button variant="primary" size="md">
  Save Changes
</Button>
```

### Variants

```jsx
<>
  <Button variant="primary">Primary</Button>
  <Button variant="secondary">Secondary</Button>
  <Button variant="ghost">Ghost</Button>
</>
```

### Sizes

```jsx
<>
  <Button size="sm">Small</Button>
  <Button size="md">Medium</Button>
  <Button size="lg">Large</Button>
</>
```

### With Icons

```jsx
<>
  <Button leadingIcon={<Search size={16} />}>
    Search
  </Button>
  <Button trailingIcon={<ArrowRight size={16} />}>
    Next Step
  </Button>
  <Button leadingIcon={<Plus size={16} />}>
    Add Item
  </Button>
</>
```

### Loading State

```jsx
<Button variant="primary" loading>
  Saving...
</Button>
```

### Disabled State

```jsx
<Button disabled>
  Disabled
</Button>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| variant | `'primary' \| 'secondary' \| 'ghost'` | `'primary'` | Visual style of the button. |
| size | `'sm' \| 'md' \| 'lg'` | `'md'` | Size of the button — affects padding and font size. |
| loading | `boolean` | `false` | When true, renders a spinner and sets aria-busy. The button remains focusable but click handlers are blocked. |
| disabled | `boolean` | `false` | When true, the button is visually and functionally disabled. |
| leadingIcon | `React.ReactNode` | — | Icon rendered to the left of the label. Marked as decorative with aria-hidden. |
| trailingIcon | `React.ReactNode` | — | Icon rendered to the right of the label. Marked as decorative with aria-hidden. |
| type | `'button' \| 'submit' \| 'reset'` | `'button'` | HTML button type attribute. Use 'submit' within a form context. |
| onClick | `(event: React.MouseEvent<HTMLButtonElement>) => void` | — | Click handler. Blocked when button is disabled or loading. |
| className | `string` | — | Additional CSS class names. |
| All HTML button attributes | — | — | Supports ref, aria-label, title, data-* attributes, etc. |

## Accessibility

- **Button type**: Defaults to `type="button"`. Use `type="submit"` on forms.
- **Disabled state**: When `disabled` is true, the HTML `disabled` attribute is set, preventing keyboard and pointer interaction.
- **Loading state**: When `loading` is true, `aria-busy="true"` is set and the button remains focusable. Click handlers are prevented via JavaScript. Icon spinners are marked `aria-hidden="true"`.
- **Icon decoration**: Both `leadingIcon` and `trailingIcon` are wrapped in `aria-hidden="true"` spans, so screen reader users only hear the button label.
- **Keyboard behavior**: All buttons receive focus in tab order. Activate with Enter or Space. When loading, the button holds focus but prevents action.
- **Focus management**: Use ref forwarding to programmatically manage focus after an action completes.

## When to use Button vs. Link

Use a **Button** when:
- The interaction changes application state (e.g., save, delete, create)
- The interaction submits a form
- The interaction opens a modal or overlay
- The interaction triggers a background process (e.g., upload, generate)

Use a **Link** (styled as a button) when:
- The interaction navigates to a different page or route
- The interaction opens a new window or external URL
- Users should be able to open the destination in a new tab via right-click

## Do / Don't

### Do

```jsx
<form>
  <input type="text" placeholder="Enter name" />
  <Button type="submit" variant="primary">
    Submit Form
  </Button>
</form>
```

Buttons should have clear, action-oriented labels and appropriate HTML type for their context.

### Don't

```jsx
<Button onClick={() => navigate('/dashboard')}>
  Go to Dashboard
</Button>
```

Don't use Button for navigation. Use a Link component instead, so users can right-click, open in new tab, and search engines can crawl the navigation.
