# Modal

## Overview

Modal is a dialog component that displays content in a floating panel above the page, with a backdrop that dims the background. It manages focus automatically, traps keyboard navigation, and closes on ESC key press or backdrop click by default. Use modals for critical user decisions, forms that require undivided attention, or alerts that demand acknowledgment before continuing.

## Usage

```jsx
import { Modal } from '@enterprise-design-system/react'
import { useState } from 'react'

export function ModalExample() {
  const [open, setOpen] = useState(false)

  return (
    <>
      <button onClick={() => setOpen(true)}>
        Open Modal
      </button>

      <Modal
        open={open}
        onClose={() => setOpen(false)}
        header="Confirm Action"
        footer={
          <div style={{ display: 'flex', gap: '8px', justifyContent: 'flex-end' }}>
            <button onClick={() => setOpen(false)}>Cancel</button>
            <button onClick={() => setOpen(false)}>Confirm</button>
          </div>
        }
      >
        <p>Are you sure you want to proceed with this action?</p>
      </Modal>
    </>
  )
}
```

## Composition

The Modal component has three main regions:

- **Header** ŌĆö Optional title bar with an auto-generated close button. When provided, the header text becomes the accessible label via `aria-labelledby`.
- **Body** ŌĆö Main content area passed as `children`.
- **Footer** ŌĆö Optional action buttons or additional controls below the body.

## Sizes

The `size` prop controls the modal width:

| Size | Width |
|------|-------|
| `sm` | 384px |
| `md` | 512px (default) |
| `lg` | 640px |

```jsx
<Modal
  open={open}
  onClose={() => setOpen(false)}
  size="lg"
  header="Large Modal"
>
  Content
</Modal>
```

## Focus Management

The Modal automatically:

1. **Captures focus** ŌĆö When opened, focus moves to the first focusable element inside the dialog, or to the dialog itself if no focusable elements exist.
2. **Traps focus** ŌĆö Pressing Tab cycles through focusable elements inside the modal only. Shift+Tab at the first element wraps to the last.
3. **Restores focus** ŌĆö When closed, focus returns to the element that was active before the modal opened.

## Closing Behavior

By default, the Modal closes when:

- The user presses **ESC**
- The user clicks the **backdrop** (dimmed area)
- The user clicks the built-in **close button** in the header

Use `disableEscClose` or `disableBackdropClose` props to prevent unintended dismissals in critical flows (e.g., confirmation dialogs).

```jsx
<Modal
  open={open}
  onClose={() => setOpen(false)}
  disableBackdropClose
  disableEscClose
  header="Confirm Deletion"
>
  <p>This action cannot be undone.</p>
</Modal>
```

## Accessibility

**ARIA Roles & Attributes:**

- `role="dialog"` ŌĆö Identifies the panel as a dialog container.
- `aria-modal="true"` ŌĆö Marks the dialog as modal, preventing interaction with the page below.
- `aria-label` or `aria-labelledby` ŌĆö Provides an accessible name. Use `aria-label` for icon-only headers; pass `header` to auto-wire `aria-labelledby`.
- `aria-hidden="true"` ŌĆö Applied to the backdrop element so it is not announced.

**Keyboard Behavior:**

- **ESC** ŌĆö Closes the modal (unless `disableEscClose` is true).
- **Tab / Shift+Tab** ŌĆö Cycles focus within the modal; wraps at edges.

**Screen Reader Notes:**

- Always provide either a `header` or an explicit `aria-label` prop so screen reader users know the dialog's purpose.
- The close button is automatically labeled with "Close modal".
- Background scrolling is disabled while the modal is open.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | ŌĆö | Controls whether the modal is visible. The parent component is responsible for updating this state when `onClose` is called. |
| `onClose` | `() => void` | ŌĆö | Callback fired when the user attempts to close the modal via ESC, backdrop click, or the close button. The parent must use this to update the `open` prop. |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Width preset for the dialog panel. |
| `disableBackdropClose` | `boolean` | `false` | When true, clicking the backdrop will not close the modal. |
| `disableEscClose` | `boolean` | `false` | When true, pressing ESC will not close the modal. |
| `aria-label` | `string` | ŌĆö | Accessible label for the dialog. Use when there is no `header`. Ignored if `header` is provided (which auto-wires `aria-labelledby`). |
| `header` | `React.ReactNode` | ŌĆö | Content rendered in the header bar. Automatically wired as the dialog's accessible label. |
| `children` | `React.ReactNode` | ŌĆö | Content rendered in the body region. |
| `footer` | `React.ReactNode` | ŌĆö | Content rendered in the footer region, typically containing action buttons. |
| `className` | `string` | ŌĆö | Additional CSS class names applied to the dialog panel. |
| `style` | `React.CSSProperties` | ŌĆö | Inline styles applied to the dialog panel. |

## Do / Don't

### Do

```jsx
// Provide a header or aria-label so the dialog has an accessible name
<Modal
  open={open}
  onClose={() => setOpen(false)}
  header="Save Changes?"
>
  <p>Discard unsaved work?</p>
  <button onClick={() => setOpen(false)}>Cancel</button>
  <button onClick={handleSave}>Save</button>
</Modal>
```

### Don't

```jsx
// Missing both header and aria-label ŌĆö screen readers cannot announce the modal's purpose
<Modal open={open} onClose={() => setOpen(false)}>
  <p>Discard unsaved work?</p>
</Modal>
```
