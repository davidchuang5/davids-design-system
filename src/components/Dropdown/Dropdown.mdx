import { Dropdown } from './index'

# Dropdown

## Overview

Dropdown is a menu component that displays a list of selectable options when triggered. Use it to let users choose from a set of items in a compact, space-saving interface. The component supports single selection, custom triggers, item icons, visual variants, and multiple placement options. It handles keyboard navigation, focus management, and click-outside behavior automatically.

## Usage

```jsx
import { Dropdown } from '@enterprise/design-system'

const items = [
  { id: '1', label: 'Profile' },
  { id: '2', label: 'Settings' },
  { id: '3', label: 'Logout' },
]

export function DropdownExample() {
  const [selectedId, setSelectedId] = React.useState('1')

  return (
    <Dropdown
      label="Choose an option"
      items={items}
      selectedId={selectedId}
      onChange={(item) => setSelectedId(item.id)}
    />
  )
}
```

## Item Structure

Each item in the `items` array follows the `DropdownItem` interface:

| Property | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string | Yes | Unique identifier for the item |
| `label` | string | Yes | Display text shown in the menu |
| `icon` | React.ReactNode | No | Optional icon rendered before the label |
| `disabled` | boolean | No | When true, item is shown but cannot be selected |
| `divider` | boolean | No | When true, renders as a visual separator (non-interactive) |

const items = [

## Placement

The `placement` prop controls where the menu opens relative to the trigger. Four options are available:

```jsx
// Opens below the trigger, aligned to the left edge
<Dropdown placement="bottom-start" items={items} label="Menu" />

// Opens below the trigger, aligned to the right edge
<Dropdown placement="bottom-end" items={items} label="Menu" />

// Opens above the trigger, aligned to the left edge
<Dropdown placement="top-start" items={items} label="Menu" />

// Opens above the trigger, aligned to the right edge
<Dropdown placement="top-end" items={items} label="Menu" />
```

## Controlled vs Uncontrolled

The Dropdown supports both controlled and uncontrolled patterns.

### Uncontrolled (default)

The component manages selection and open state internally. Use for simpler use cases:

```jsx
<Dropdown
  label="Select"
  items={items}
  defaultSelectedId="1"
  defaultOpen={false}
  onChange={(item) => console.log('Selected:', item)}
/>
```

### Controlled

Provide `selectedId`, `open`, `onChange`, and `onOpenChange` to manage state externally:

```jsx
const [selectedId, setSelectedId] = React.useState('1')
const [isOpen, setIsOpen] = React.useState(false)

<Dropdown
  label="Select"
  items={items}
  selectedId={selectedId}
  onChange={(item) => setSelectedId(item.id)}
  open={isOpen}
  onOpenChange={setIsOpen}
/>
```

## Keyboard Interactions

| Key | Context | Behavior |
| --- | --- | --- |
| **Enter** | Trigger focused | Opens menu, focus moves to first item |
| **Space** | Trigger focused | Opens menu, focus moves to first item |
| **ArrowDown** | Trigger focused | Opens menu, focus moves to first item |
| **ArrowUp** | Trigger focused | Opens menu, focus moves to last item |
| **Escape** | Menu open | Closes menu, focus returns to trigger |
| **ArrowDown** | Item focused | Moves focus to next navigable item |
| **ArrowUp** | Item focused | Moves focus to previous navigable item |
| **Home** | Item focused | Moves focus to first navigable item |
| **End** | Item focused | Moves focus to last navigable item |
| **Enter** | Item focused | Selects item, closes menu, focus returns to trigger |
| **Space** | Item focused | Selects item, closes menu, focus returns to trigger |
| **Tab** | Menu open | Closes menu, focus returns to trigger |

Disabled and divider items are skipped during keyboard navigation.

## Accessibility

- **ARIA roles**: The trigger uses `aria-haspopup="listbox"` and `aria-expanded` to announce state. The menu has `role="listbox"` with items as `role="option"`.
- **Focus management**: Focus is automatically moved to the trigger after selection and when closing the menu.
- **Screen reader announcements**: Selected items display a checkmark and are marked with `aria-selected="true"`. Disabled items are marked with `aria-disabled="true"`.
- **Labeling**: The menu is labeled via `aria-labelledby` pointing to the trigger. Override with `menuLabel` if needed.
- **Visual indicators**: Dividers use `role="separator"` and `aria-hidden="true"` to hide from screen readers.

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `label` | string | Required | Trigger button text (used when no custom trigger provided) |
| `items` | DropdownItem[] | Required | Array of menu items to display |
| `selectedId` | string | undefined | Controlled selected item id |
| `defaultSelectedId` | string | undefined | Default selected item id (uncontrolled) |
| `onChange` | (item: DropdownItem) => void | undefined | Callback fired when an item is selected |
| `open` | boolean | undefined | Controlled open state |
| `defaultOpen` | boolean | false | Default open state (uncontrolled) |
| `onOpenChange` | (open: boolean) => void | undefined | Callback fired when open state changes |
| `size` | 'sm' \| 'md' \| 'lg' | 'md' | Visual size of trigger and items |
| `variant` | 'default' \| 'outline' \| 'ghost' | 'default' | Visual style of the trigger button |
| `placement` | 'bottom-start' \| 'bottom-end' \| 'top-start' \| 'top-end' | 'bottom-start' | Menu placement relative to trigger |
| `disabled` | boolean | false | Disables the entire dropdown |
| `trigger` | React.ReactElement | undefined | Custom trigger element (receives ARIA props via clone) |
| `menuLabel` | string | undefined | Accessible label for the listbox (defaults to `label`) |
| `className` | string | undefined | Additional CSS class for the root container |
| `...divProps` | HTMLAttributes<HTMLDivElement> | | Standard HTML div attributes |

## Do / Don't

### Do

```jsx
// Provide meaningful labels and use clear item text
<Dropdown
  label="Export format"
  items={[
    { id: 'csv', label: 'CSV' },
    { id: 'json', label: 'JSON' },
    { id: 'pdf', label: 'PDF' },
  ]}
  onChange={(item) => exportAs(item.id)}
/>
```

### Don't

```jsx
// Don't use vague labels or hide important context
<Dropdown
  label="Choose"
  items={[
    { id: '1', label: 'Option 1' },
    { id: '2', label: 'Option 2' },
  ]}
/>

// Don't render interactive elements inside items (use label and icon only)
<Dropdown
  label="Menu"
  items={[
    {
      id: 'action',
      label: 'Action',
      icon: <button>Click me</button>, // Don't do this
    },
  ]}
/>
```
